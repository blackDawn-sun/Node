## 1. Nous Research 公司介绍

### 1.1 创立与理念

**Nous Research** 是一家开源 AI 研究实验室，2023 年成立于美国德克萨斯州奥斯汀。最初的成员是一群在社交媒体上结识的志愿者 AI 研究员，他们怀着一个共同的信念：打造透明、无限制的开源 AI，挑战 OpenAI、Anthropic 等闭源巨头的垄断格局。

公司名称 **"Nous"** 源自古希腊哲学概念 **νοῦς**（nous），意为"直觉"或"心智"——这个名字准确地传达了他们的使命：创造真正理解人类意图、服务于个体自由的智能。

> *"超级智能应当服务于最大化的个体自主性和精神自由。它的发展不能只掌握在少数公司和寡头手中。"*
> — Nous Research 官方宣言

Nous Research 的核心理念可以概括为三个关键词：

- **开源（Open Source）**：所有模型、代码、数据集公开，社区可自由使用、修改、分发
- **无限制（Unrestricted）**：模型拒绝率仅约 40%（商业模型通常 95% 以上），最大限度保留模型的回答能力
- **人类对齐（Human-Aligned）**：不是"安全对齐"（拒绝回答），而是让模型忠实执行用户意图

### 1.4 产品矩阵

#### Hermes 模型系列

Hermes 是 Nous Research 的旗舰开源大语言模型系列，基于 Llama 等基础架构进行高质量合成数据微调。超过 **90% 的训练数据是合成的**，通过内部 DataForge 管线生成，让一个小团队能在数据质量上媲美大厂。

| 模型 | 基础架构 | 核心特点 |
|------|---------|---------|
| Hermes 2 Pro | Llama-3 | 高质量指令微调、函数调用 |
| Hermes 2 Theta | Llama-3 | 增强推理、代码生成 |
| Hermes 3 | Llama 3.1 70B | 更高表现力、长链思考、个性化对齐 |
| Hermes 4 / 4.3 | 最新 | 90% 合成数据；~60% 非拒绝率；MATH 超 Meta Llama Instruct 约 60 分 |
| DeepHermes-3 | — | 快速响应与思维链推理两种模式自由切换 |
| NousCoder-14B | — | 代码竞赛型模型，LiveCodeBench v6 Pass@1 达 67.87% |

#### DisTrO — 分布式训练优化器

**DisTrO**（Distributed Training Over-the-Internet Optimizer）是 Nous Research 自研的分布式训练协议，能将训练过程中的数据传输量降低 **3~4 个数量级**（最高 10,000 倍效率提升）。它使用 **DeMo**（Decoupled Momentum Optimization，解耦动量优化）算法，在保持模型精度的同时大幅降低带宽需求，让跨地域、消费级网络环境下的分布式训练成为现实。

2024 年 12 月，团队**直播演示**了用纯分布式、去中心化硬件成功预训练一个 15 亿参数模型，证明了技术可行性。

#### Psyche Network — 去中心化训练网络

Psyche 是构建在 **Solana 区块链**上的去中心化 AI 训练基础设施。它协调全球贡献者的分布式 GPU 算力（从 NVIDIA 4090 到 A100、H100），通过代币激励模型奖励算力提供者。目前处于测试阶段，已开始在 Seed-OSS-36B 模型上进行 Hermes 4 的消融实验和训练。

## 2. Hermes Agent 项目概述

> 官方文档：<https://hermes-agent.nousresearch.com/docs>
> GitHub：<https://github.com/NousResearch/hermes-agent>

### 2.1 什么是 Hermes Agent

Hermes Agent 是由 Nous Research 开发的开源 AI Agent 框架。它可以运行在终端、桌面应用、消息平台（Telegram、Discord、Slack 等 23+ 个平台）中，自主调用工具完成任务。

**本质上一个拥有内置学习循环的自主 AI Agent**：它会从经验中创造技能、在使用过程中自我改进、搜索自己的历史对话获取上下文、并跨会话建立你的用户画像。

它可以跑在一台 $5 的 VPS 上、GPU 集群里，或者按需启停的无服务器基础设施上（Modal、Daytona）。它**不绑定在你的笔记本上**——你可以从 Telegram 跟它聊天，而它正在一台云 VM 上工作。

### 2.2 发展历程

| 时间 | 里程碑 |
|------|--------|
| 2024 年中 | Hermes Agent 项目启动 |
| 2025.2 | 首次开源发布，GitHub 首月获 2.2K stars |
| 2025.4 | 从 OpenClaw 推出一键迁移工具，抓住 Anthropic 政策变化带来的用户迁移潮 |
| 2026.4 | v0.10.0，引入 Tool Gateway，发布 400+ 模型 |
| 2026.5 | v0.13~0.15 密集迭代：Kanban 持久化、TUI、代码重构、Swarm |
| 2026.6 | v0.16.0 "Surface"，Hermes Desktop 公测，原生桌面体验 |

GitHub Stars 超过 **110K**，是目前最活跃的开源 Agent 项目之一。

### 2.3 核心亮点

- **技能自进化**：Agent 解决复杂问题后，会将可复用流程保存为 Skill 文件。Skill 是透明的、人类可读可编辑的。随着时间推移，Agent 构建起一套针对你个人用例的技能库
- **跨会话持久记忆**：不像无状态 Agent 每次对话从零开始，Hermes 跨会话保存关键信息，建立个性化用户画像——用得越多越聪明
- **多平台 Gateway**：同一个 Agent 在 Telegram、Discord、Slack、微信等 23+ 个平台上运行，拥有完整工具访问权限
- **Profile 多实例**：运行多个互相隔离的 Hermes 实例，各有独立的配置、会话、技能和记忆
- **可扩展**：插件系统、MCP 服务器、自定义工具、Webhook 触发、定时任务
- **自维护 Curator**：后台自动审查 Agent 创建的本地址能，合并重叠、归档过时条目，防止技能库无限膨胀

### 2.4 与 OpenClaw 的对比

Hermes Agent 经常被拿来与 OpenClaw（"龙虾"）对比。两者都提供类似的功能，但存在关键差异：

| 维度 | Hermes Agent | OpenClaw |
|------|-------------|----------|
| 技能存储 | 透明、人类可读的 SKILL.md 文件 | 不透明格式 |
| 自我改进 | 内置 Curator 自动维护技能库 | 无 |
| Provider 灵活性 | 20+ 提供商，400+ 模型 | 较少 |
| Windows 支持 | v0.14+ 原生支持 | 有限 |
| 桌面应用 | v0.16+ Hermes Desktop | 无 |
| 许可证 | MIT | — |

## 3. 快速上手

> 官方文档：<https://hermes-agent.nousresearch.com/docs/getting-started/installation>

### 3.1 环境要求

- **Python**：3.11+
- **操作系统**：Linux、macOS、WSL2、Windows（v0.14+ 原生）、Android Termux
- **Git**：安装脚本依赖

### 3.2 安装

安装前请确认 git 可用。

安装脚本主要会做这些事：

- 拉取或更新 Hermes 源码。普通 Linux / macOS / WSL2 用户默认安装到 `~/.hermes/hermes-agent`；Windows 默认安装到 `%LOCALAPPDATA%\hermes\hermes-agent`
- 创建 Python 虚拟环境并安装依赖；必要时会安装 `uv`、Node.js、浏览器工具相关依赖
- 创建 `hermes` 命令入口，并提示把它所在目录加入 `PATH`
- 初始化数据目录。Linux / macOS / WSL2 默认是 `~/.hermes`；Windows 默认是 `%LOCALAPPDATA%\hermes`
- 仅在文件不存在时创建 `config.yaml`、`.env`、`SOUL.md`，已有配置会保留
- 交互式终端中会继续运行 setup 向导

#### Linux / macOS / WSL2

```bash
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
```

#### Windows（PowerShell，v0.14+ 原生支持）

```powershell
iex (irm https://hermes-agent.nousresearch.com/install.ps1)
```

Windows 原生安装使用便携版 Git Bash，无需 WSL。安装后会在 `%LOCALAPPDATA%\hermes\` 下创建完整运行环境。

#### Termux (Android)

```bash
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
```

v0.15 冷启动优化后，Termux 环境下 `hermes --version` 仅需 **0.8 秒**。

### 3.3 初次配置

```bash
hermes model                 # 交互式选择模型和提供商
hermes setup                 # 或者运行完整设置向导
hermes setup --portal        # 使用 Nous Portal 一键配置（推荐新用户）
```

**Nous Portal** 是 v0.16 推荐的一键配置方式，提供 400+ 模型，OAuth 登录后即可使用，无需手动管理 API Key。

API Key 存储在 `~/.hermes/.env` 文件中（Windows 在 `%LOCALAPPDATA%\hermes\.env`）。

### 3.4 基本使用

#### Hermes Desktop（v0.16+，★ 推荐新用户）

Hermes Desktop 是 v0.16 推出的原生桌面应用（基于 Electron），macOS、Linux、Windows 全平台支持。它是对 CLI 和 TUI 模式的补充——**不需要打开终端**，拥有完整的图形化界面和中文支持。

**安装**：从 [Hermes Desktop 公测页面](https://hermes-agent.nousresearch.com/docs/getting-started/desktop) 下载对应平台的安装包，一键安装。应用内支持自动更新。

**启动后的第一个界面就是模型配置**：

- 支持模糊搜索模型（输入 `sonnet` 瞬间匹配到 `anthropic/claude-sonnet-4`）
- 内置 Nous Portal OAuth 登录，400+ 模型一键切换
- 所有主流 Provider 均可通过图形界面配置 API Key

**核心功能**：

| 功能 | 说明 |
|------|------|
| 多标签页对话 | 同时进行多个独立会话，各自拥有独立的上下文 |
| Cmd/Ctrl+K 命令面板 | 类似 VS Code，搜索和执行任意操作 |
| 拖放文件 | 将文件拖入聊天窗口即可让 Agent 处理 |
| 剪贴板图片粘贴 | 截图后 Ctrl+V 直接粘贴到对话中 |
| Remote Gateway | 桌面应用通过 OAuth 连接到运行在云服务器上的 Hermes Gateway |
| 完整中文界面 | v0.16 正式支持简体中文 UI |

**与 CLI/TUI 的关系**：Hermes Desktop 底层仍然调用同一个 `hermes` 引擎，所有配置、会话、技能、记忆与 CLI/TUI 模式完全共享。你可以白天在 Desktop 上聊天，晚上在终端里跑自动化 Cron 任务——使用的是同一套数据。

![Hermes Desktop 模型切换](images/hermes%20desktop模型切换.png)

#### 交互式对话（终端）

```bash
hermes                       # 启动交互式对话
hermes --tui                 # 使用 TUI 启动（v0.11+，React/Ink 终端界面）
```

**TUI 模式**（v0.11+）提供：
- 实时 token 流式输出
- 每轮对话耗时统计
- 子 Agent 可观测性面板
- 多会话管理
- 固定输入栏（类似聊天应用）

#### 单次对话（非交互式）

```bash
hermes chat -q "查看系统资源占用情况"
```

适合脚本调用或一次性任务。

#### 诊断

```bash
hermes doctor                # 检查环境健康状态，诊断潜在问题
```

| `Ctrl + C` | 中断当前操作 |

### 3.6 更新

```bash
hermes update
```

> **建议**：升级前备份 `~/.hermes/auth.json` 和 `~/.hermes/config.yaml`。大版本升级时建议直接跳到最新补丁版本（如 v0.15.1 而非 v0.15.0）。

### 3.7 卸载

```bash
hermes uninstall
```

如需完全清理所有数据：

```bash
# Linux / macOS / WSL2
rm -rf ~/.hermes

# Windows
Remove-Item -Recurse -Force $env:LOCALAPPDATA\hermes
```

## 4. 配置文件与配置管理

> 官方文档：<https://hermes-agent.nousresearch.com/docs/user-guide/configuration>

### 4.1 配置目录结构

```text
~/.hermes/
├── config.yaml        # 主配置文件（模型、终端、TTS、压缩等）
├── .env               # API 密钥和机密信息
├── auth.json          # OAuth 提供商凭证（Nous Portal 等）
├── SOUL.md            # 主 Agent 的身份 / 人格文件，会拼入系统提示词开头部分
├── memories/          # 持久化记忆（MEMORY.md、USER.md）
├── skills/            # 技能
├── cron/              # 定时任务
├── sessions/          # 会话
└── logs/              # 日志（errors.log、gateway.log — 密钥自动脱敏）
```

### 4.2 配置管理命令

```bash
hermes config                        # 查看当前配置
hermes config edit                   # 用 $EDITOR 打开 config.yaml 编辑
hermes config set section.key value  # 直接设置某个配置项
```

会话内也可以通过斜杠命令调整部分配置：

| 命令 | 功能 |
|------|------|
| `/config` | 查看当前配置 |
| `/model [model-name]` | 查看或切换当前会话使用的模型 |
| `/personality [name]` | 切换预设交互风格 / 人格 |
| `/reasoning [level/show/hide]` | 调整或查看模型推理级别 |
| `/voice [on/off/tts/status]` | 开启、关闭或查看语音输入 / TTS 输出状态 |
| `/yolo` | 切换绕过确认模式，减少工具执行确认；只建议在可信环境和低风险任务使用 |

### 4.3 Provider 体系详解

Hermes 支持 **20+ 模型提供商**，通过 `hermes model` 交互切换，无需修改代码。这是 Hermes 区别于大多数 Agent 框架的核心优势之一——完全不被任何单一模型厂商锁定。

#### 主要提供商一览

| 提供商 | 接入方式 | 特点 |
|--------|---------|------|
| **Nous Portal** | OAuth | 首选；400+ 模型，统一计费 |
| **Anthropic** | API Key / OAuth PKCE | Claude 系列；自动发现 Claude Code 凭证；支持 Prompt Caching |
| **OpenAI** | API Key | GPT-5 系列，Chat Completions + Codex Responses API |
| **OpenRouter** | API Key | 200+ 模型统一网关 |
| **AWS Bedrock** | IAM | 原生 Converse API；模型发现；Guardrails |
| **Google Gemini** | CLI OAuth | 通过 Google 账号授权 |
| **xAI / Grok** | API Key / OAuth Proxy | Web Search 插件；TTS 语音合成；模型退役自动检测 |
| **DeepSeek** | API Key | 高性价比 |
| **GitHub Copilot** | OAuth | 400K 上下文窗口 |
| **Hugging Face** | API Key | HF Inference API，精选 Agent 模型选择器 |
| **NVIDIA NIM** | API Key | 托管或自托管端点 |
| **LM Studio** | 本地 | 原生本地提供商 |
| **MiniMax、Kimi/Moonshot、z.ai/GLM** | API Key | 国内厂商 |
| **自定义端点** | API Key | 任何 OpenAI 兼容 API |
